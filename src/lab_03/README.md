# Лабораторная работа №3: Наследование и иерархия классов

## Цель работы
- Освоить механизм наследования классов в Python.
- Научиться строить иерархии объектов (базовый класс – производные классы).
- Переиспользовать код базового класса с помощью `super()`.
- Освоить переопределение методов и полиморфизм.
- Понять, как полиморфизм позволяет единообразно работать с объектами разных типов.

---

## Описание реализованной иерархии классов

**Предметная область:** Библиотека / Книги (вариант 2).  
На основе базового класса `Book` (из ЛР-1) построена иерархия из трёх дочерних классов.

### Базовый класс `Book` (абстрактный)
- **Атрибуты:** `_title`, `_author`, `_year`, `_pages`, `_is_loaned` (защищённые).
- **Свойства:** `title`, `author`, `year`, `pages`, `is_loaned` (геттеры/сеттеры с валидацией).
- **Бизнес-методы:** `issue()`, `return_book()` – меняют статус выдачи.
- **Абстрактный метод:** `get_reading_time()` – объявляет интерфейс для всех книг (возвращает предполагаемое время чтения в минутах). В базовом классе не реализован, что заставляет наследников предоставить свою реализацию.
- **Магические методы:** `__str__`, `__repr__`, `__eq__`.

Базовый класс объявлен **абстрактным** (наследует от `ABC` и содержит `@abstractmethod`). Это означает, что нельзя создать объект `Book` напрямую – только через его наследников. Такой подход гарантирует, что каждый конкретный вид книги обязательно реализует метод `get_reading_time()`.

### Производные классы

#### 1. `PrintedBook` (печатная книга)
- **Добавленные атрибуты:**
  - `cover_type` – тип обложки (`"твердая"` или `"мягкая"`)
  - `is_illustrated` – наличие иллюстраций (булево)
- **Реализация `get_reading_time`:** время чтения = количество страниц × 2 минуты (средняя скорость чтения печатной книги).

#### 2. `Ebook` (электронная книга)
- **Добавленные атрибуты:**
  - `file_size` – размер файла в мегабайтах
  - `file_format` – формат (`pdf`, `epub`, `mobi` и т.п.)
- **Реализация `get_reading_time`:** время = страницы × 1.5 минуты (электронные книги читаются немного быстрее из-за удобства навигации).

#### 3. `AudioBook` (аудиокнига)
- **Добавленные атрибуты:**
  - `duration` – длительность в минутах
  - `narrator` – имя чтеца
- **Реализация `get_reading_time`:** возвращает `duration` (время прослушивания не зависит от количества страниц).

### Переопределение методов
Все три класса переопределяют:
- `get_reading_time()` – для полиморфного расчёта времени.
- `__str__()` – добавляют к базовому выводу информацию о специфических атрибутах (например, `[Печатная, обложка: твердая]`).

### Интеграция с коллекцией (ЛР-2)
Класс `Library` (коллекция книг) был расширен методами фильтрации по типу:
- `get_printed_books()`
- `get_ebooks()`
- `get_audiobooks()`

Эти методы возвращают новую коллекцию, содержащую только объекты указанного типа. Фильтрация использует `isinstance()` для проверки принадлежности к классу.

## Вывод

В ходе лабораторной работы были изучены и применены на практике:

### 1. Наследование
- Создан абстрактный базовый класс `Book`, который содержит общие поля и методы для всех видов книг.
- Три дочерних класса (`PrintedBook`, `Ebook`, `AudioBook`) унаследовали всю функциональность базового класса и добавили свои уникальные атрибуты.
- Использование `super().__init__(...)` позволило не дублировать код инициализации общих полей.

### 2. Полиморфизм
- Абстрактный метод `get_reading_time()` объявлен в базовом классе, а каждый наследник реализует его по-своему.
- В демонстрации цикл `for book in library:` вызывает `book.get_reading_time()`, и в зависимости от реального типа книги выполняется нужная версия метода – без использования `if type == ...`.
- Переопределение `__str__` также демонстрирует полиморфизм: при печати объекта вызывается переопределённая версия, которая добавляет специфичную информацию.

### 3. Инкапсуляция и абстракция
- Базовый класс `Book` скрывает свои внутренние поля (`_title`, `_is_loaned`) и предоставляет доступ через свойства (`@property`).
- Абстрактный класс задаёт **интерфейс** (метод `get_reading_time`), который должны реализовать все наследники. Это позволяет писать код, работающий с любым объектом, производным от `Book`, не заботясь о его конкретном типе.

### 4. Работа с коллекцией разнородных объектов
- Класс `Library` из ЛР-2 был расширен методами фильтрации по типу (`get_printed_books()` и т.д.). Это показывает, как можно управлять группой объектов, принадлежащих разным классам одной иерархии.
- Благодаря полиморфизму коллекция может хранить любые объекты, наследующие `Book`, и вызывать у них общие методы (например, `issue()`, `get_reading_time()`).

### 5. Проверка типов
- Использование `isinstance()` позволяет в нужный момент определить конкретный тип объекта и выполнить специфичные действия (например, при фильтрации). Это не нарушает принципов ООП, если используется для **классификации**, а не для подмены полиморфизма.

Таким образом, лабораторная работа закрепила понимание основных механизмов объектно-ориентированного программирования: наследования, полиморфизма, абстракции и инкапсуляции. Код стал гибким, легко расширяемым (можно добавить новый вид книги, не меняя существующие классы и коллекцию) и соответствует принципу открытости/закрытости (OCP).
