# Лабораторная работа №4: Интерфейсы и абстрактные классы (ABC)

## 1. Цель работы
- Познакомиться с **абстрактными базовыми классами (ABC)** в Python.
- Освоить понятие **интерфейса** – контракта, который обязывает классы реализовывать определённые методы.
- Научиться задавать **обязательные методы** для классов через абстрактные методы.
- Закрепить **полиморфизм** через единый интерфейс (работа с объектами разных типов через общие методы).
- Спроектировать архитектуру, где поведение классов определяется интерфейсами, а не конкретными типами.

## 2. Описание интерфейсов

Создано **два интерфейса** (абстрактных класса) в файле `interfaces.py`:

### `Printable` (представление в виде строки)
- **Абстрактный метод:** `to_string() -> str`
- **Назначение:** обязать класс уметь возвращать своё строковое представление (для вывода на экран, логирования и т.п.).

### `Comparable` (сравнение объектов)
- **Абстрактный метод:** `compare_to(other) -> int`
- **Назначение:** обязать класс уметь сравнивать себя с другим объектом.
- **Возвращаемое значение:**
  - отрицательное число, если `self < other`
  - `0`, если `self == other`
  - положительное число, если `self > other`

Оба интерфейса наследуют от `ABC` (абстрактный базовый класс) и используют декоратор `@abstractmethod` для объявления обязательных методов.

## 3. Реализация интерфейсов в классах

Интерфейсы реализованы в иерархии классов **«Книги»** (из предыдущих лабораторных).

### Базовый класс `Book`
- **Реализует** `Printable` и `Comparable`.
- **Метод `to_string()`:** возвращает строку в формате:  
  `«Название» Автор (Год) – статус (доступна/выдана)`.
- **Метод `compare_to(other)`:** сравнивает книги по **году издания** (разница `self.year - other.year`).

### Дочерние классы: `PrintedBook`, `Ebook`, `AudioBook`
- **Наследуют** реализацию интерфейсов от `Book`.
- **Переопределяют** метод `to_string()` для добавления специфической информации:
  - `PrintedBook`: добавляет тип обложки и наличие иллюстраций.
  - `Ebook`: добавляет формат и размер файла.
  - `AudioBook`: добавляет длительность и имя чтеца.
- **Метод `compare_to()`** не переопределяют (остаётся сравнение по году).

**Поведение методов разное:**  
- `to_string()` у каждого класса выводит свои детали, что демонстрирует полиморфизм.

## Вывод

В ходе работы были созданы два интерфейса (`Printable`, `Comparable`) с помощью модуля `abc`. Классы книг реализовали эти интерфейсы, что позволило:

- **Писать универсальные функции** (`print_all`, `sort_by_comparable`), работающие с любыми объектами, поддерживающими интерфейс.
- **Фильтровать коллекцию по интерфейсу** (`get_printable`, `get_comparable`).
- **Использовать полиморфизм без проверки конкретных типов** – вызов `to_string()` или `compare_to()` сам определяет нужное поведение.

Такой подход соответствует принципам объектно-ориентированного проектирования (открытости/закрытости, подстановки Лисков) и облегчает расширение системы (можно добавить новый класс, реализующий эти интерфейсы, и он сразу будет работать со всеми универсальными функциями).
