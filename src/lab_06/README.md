# Лабораторная работа №6: Generics и typing

## 1. Цель работы
- Освоить систему аннотаций типов в Python (модуль `typing`).
- Научиться создавать обобщённые (generic) классы с помощью `TypeVar` и `Generic`.
- Понять концепцию структурной типизации через `typing.Protocol`.

## 2. Описание реализованных типов и контейнеров

### Generic-класс `TypedCollection`
- Параметризован типом `T` (может быть любым).
- Внутреннее хранилище: `list[T]`.
- Реализованы методы: `add`, `remove`, `get_all`, `find`, `filter`, `map`.
- Метод `map` использует второй `TypeVar R`, что позволяет менять тип результата.

### Протоколы (структурные интерфейсы)
- **`Displayable`** – требует наличия метода `display() -> str`.
- **`Scorable`** – требует наличия метода `score() -> float`.

Классы `Book`, `PrintedBook`, `Ebook`, `AudioBook` **не наследуют** эти протоколы, но содержат соответствующие методы. Благодаря этому они автоматически соответствуют протоколам (утиная типизация).

### TypeVar с ограничениями
- `D = TypeVar('D', bound=Displayable)` – может быть только тип, у которого есть `display()`.
- `S = TypeVar('S', bound=Scorable)` – только типы с методом `score()`.

Это позволяет создавать коллекции, которые гарантированно содержат объекты с нужным поведением.

## 4. Вывод
В ходе работы изучены:
- **Аннотации типов** – делают код самодокументированным и помогают IDE.
- **Generic и TypeVar** – позволяют создавать переиспользуемые коллекции, сохраняющие информацию о типе элементов.
- **Protocol** – обеспечивает структурную (утиную) типизацию без явного наследования.
- Методы `find`, `filter`, `map` демонстрируют функциональный стиль работы с коллекциями и гибкость типов.
