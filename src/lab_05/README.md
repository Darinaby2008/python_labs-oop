# Лабораторная работа №5: Функции как аргументы. Стратегии и делегаты

## 1. Цель работы
- Освоить передачу функций как аргументов в другие функции и методы.
- Научиться применять встроенные функции высшего порядка: `map`, `filter`, `sorted`.
- Понять концепцию паттерна «Стратегия» и реализовать его на Python.
- Освоить `lambda`-выражения и их практическое применение.
- Интегрировать функциональный стиль с объектно-ориентированным кодом из предыдущих лабораторных.

## 2. Реализованные функции и стратегии

Все функции вынесены в отдельный модуль `strategies.py`.

### Стратегии сортировки (key-функции)
- `by_title` – сортировка по названию книги.
- `by_author` – сортировка по автору.
- `by_year` – сортировка по году издания.
- `by_pages` – сортировка по количеству страниц.
- `by_year_then_title` – комбинированная сортировка: сначала по году, затем по названию.

### Функции-фильтры (предикаты)
- `is_available` – отбирает книги, которые не выданы.
- `is_printed` – отбирает только печатные книги (`PrintedBook`).
- `is_ebook` – отбирает только электронные книги (`Ebook`).
- `is_audiobook` – отбирает только аудиокниги (`AudioBook`).

### Фабрика функций (замыкание)
- `make_year_filter(min_year, max_year)` – возвращает функцию-предикат, которая проверяет, попадает ли год книги в заданный диапазон. Пример: `filter_1860_1880 = make_year_filter(1860, 1880)`.

### Функции для `map` (преобразование)
- `to_short_string` – возвращает строку вида `"Название (Год)"`.
- `to_title_author` – возвращает строку `"Автор: Название"`.

### Функции для `apply` (изменение состояния)
- `add_star_to_title` – добавляет звёздочку в начало названия.
- `mark_as_rare` – добавляет метку `[Rare]` к названию.

### Паттерн «Стратегия» через callable-объекты
- `DiscountStrategy(percent)` – класс, экземпляры которого при вызове уменьшают количество страниц книги на заданный процент.
- `AddPrefixStrategy(prefix)` – класс, добавляющий заданный префикс к названию книги.

### Методы класса `Library` (расширение)
- `sort_by(key_func, reverse=False)` – сортирует текущую коллекцию по ключу, возвращает `self` (для цепочек).
- `filter_by(predicate)` – возвращает новую коллекцию, содержащую только элементы, удовлетворяющие предикату.
- `apply(func)` – применяет функцию к каждому элементу коллекции (изменяет объекты), возвращает `self`.
- `map_to(func)` – возвращает список результатов применения функции к каждому элементу (без изменения коллекции).

## 3. Демонстрация работы

### Сценарий 1: Полная цепочка `filter → sort → apply`
![Сценарий 1](images/lab05/scenario1.png)

- Создаётся коллекция из 6 книг разных типов.
- Применяется фильтр `is_available` – остаются только доступные книги.
- Затем сортировка по году (`by_year`).
- Наконец, `apply(add_star_to_title)` добавляет звёздочку к названию каждой книги.
- На каждом этапе выводится состояние коллекции.

### Сценарий 2: Замена стратегии без изменения кода коллекции
![Сценарий 2](images/lab05/scenario2.png)

- Одна и та же коллекция сортируется тремя разными стратегиями: по названию, по количеству страниц, комбинированной (год + название).
- Фильтрация разными предикатами: только электронные книги, только печатные книги.
- Использование фабрики `make_year_filter` для фильтрации по диапазону годов (1860–1880).

### Сценарий 3: Callable-объект как стратегия
![Сценарий 3](images/lab05/scenario3.png)

- Создаётся небольшая коллекция книг.
- Применяется стратегия `DiscountStrategy(20)` – уменьшает количество страниц каждой книги на 20%.
- Затем применяется стратегия `AddPrefixStrategy("[NEW] ")` – добавляет префикс к названию.
- Дополнительно показано применение `map_to(to_short_string)` для преобразования коллекции в список коротких строк.

## 4. Вывод

В ходе лабораторной работы были изучены и применены на практике:

- **Передача функций как аргументов** – функции сортировки, фильтрации и преобразования передаются в методы коллекции.
- **Lambda-выражения** – использованы для создания простых функций «на лету» (например, в `sort_by` можно передать `lambda b: b.title`).
- **Функции высшего порядка** – `map`, `filter`, `sorted` (через встроенные методы коллекции).
- **Замыкания (фабрики функций)** – `make_year_filter` демонстрирует создание функции с параметризованным условием.
- **Паттерн «Стратегия»** – реализован через callable-объекты (`DiscountStrategy`, `AddPrefixStrategy`), что позволяет легко заменять алгоритм обработки без изменения кода коллекции.
- **Цепочки операций** – методы `filter_by().sort_by().apply()` обеспечивают выразительный и компактный синтаксис.
